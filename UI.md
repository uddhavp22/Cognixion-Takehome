# Converse — UI Reference

A hover-driven recipe explorer. Every interaction is a **dwell-to-commit**: hover a tile, watch the fill ring sweep to 100 %, then the action fires. No clicks required. This is a stand-in for BCI / gaze input.

---

## Interaction model

| Term | Meaning |
|------|---------|
| **Dwell** | Hold the cursor over a tile for the configured dwell window (default 0.8 s, 1.0 s on the map) |
| **Fill ring** | Blue radial-360 arc that sweeps from 0 → 100 % as dwell progresses |
| **Warm flash** | Tile background shifts at 60 % progress to signal commitment is near |
| **Commit flash** | Brief scale-punch + color change when the dwell threshold is reached |
| **autoRearm** | After committing, tile resets to idle (used on toggles). One-shot tiles stay committed. |

All tile behavior lives in `DwellTile.cs`. The singleton `MouseDwellInput` tracks which tile is hovered, accumulates time, and broadcasts `OnCommit(tileId)`.

## Scenes

### 01 — Constraints

**File:** `Scripts/Scenes/ConstraintsSceneBootstrap.cs`
**Purpose:** Collect user preferences before generating a recipe.

**Layout:** Single centered white card (1440 × 820 px) on a dark `#0D1117` background. All content is a flat `VerticalLayoutGroup` — no nested layout groups — to avoid Unity's ContentSizeFitter double-pass problem.

**Sections (top to bottom):**

| Section | Control type | Notes |
|---------|-------------|-------|
| **Header** | Static labels | Title + subtitle in dark text on the white card |
| **Dietary Needs** | Chip pills (toggle) | 9 chips in one row, each expands equally. Multi-select; chips toggle blue when active. Checked state shown with "✓ " prefix. |
| **Serving Size** | Segmented control | 4 options: 2 / 4 / 6 / 8. Single-select; selected tile turns blue. Default: 4. |
| **Cook Time** | Segmented control | 4 options: Quick / Medium / Leisurely / Any. Default: Any. |
| **Spice Level** | Segmented control | 3 options: Mild / Medium / Spicy. Default: Medium. |
| **Continue** | Full-width CTA button | Blue `#2563EB` pill. Loads `02_Map`. |

**State written:** `AppSession.SelectedConstraints`, `SelectedServings`, `SelectedTimePreference`, `SelectedSpiceLevel`.

**Design tokens (local to this scene):**
- Card background: `#F8FAFD`
- Card border: `#CBD5E0`
- Chip unselected: `#EDF2F7`
- Chip selected / CTA: `#3B82F6` / `#2563EB`
- Text primary: `#1A202C`, secondary: `#4A5568`, muted: `#718096`

### 02 — Map

**Files:** `Scripts/Scenes/MapSceneBootstrap.cs` (stub), `Scripts/Map/WorldMapController.cs` (real map)

**Purpose:** Let the user choose a country. Selecting a country determines the cuisine origin for mood and recipe generation.

**Two implementations run in the same scene:**

#### WorldMapController (primary)
Loads `StreamingAssets/world.geojson` (Natural Earth 110m, 177 countries) and renders each country as a triangulated mesh via `EarClipping.cs`. Uses an orthographic camera sized so the full ±180° longitude is always visible.

| State | Color |
|-------|-------|
| Default | `#293F33` (muted forest green) |
| Hovered | Violet `(0.48, 0.34, 0.72)` |
| 60 % dwell warm | Rose `(0.92, 0.38, 0.62)` |

On hover, a floating **Info Panel** (300 × 120 px) follows the cursor and shows the country name. After a 500 ms debounce, it requests a one-sentence blurb from Claude (`BlurbSystem` prompt) and populates the second text line. Blurbs are cached per ISO code so re-hovering is instant.

Dwell window on this scene is **1.0 s** (longer than other scenes — hovering a country on a map needs to feel intentional).

**Back button** — bottom-left pill, dwell to return to `01_Constraints`. Fill ring uses the global blue accent.

#### MapSceneBootstrap (fallback stub)
If `world.geojson` is absent the controller falls back to 6 hardcoded bounding-box quads: Morocco, Japan, Mexico, India, Italy, Peru.

**State written:** `AppSession.SelectedCountry` (name + ISO-3 code).

### 03 — Mood

**File:** `Scripts/Scenes/MoodSceneBootstrap.cs`
**Purpose:** Narrow the recipe direction by choosing a dining mood / vibe.

**Layout:** Dark background (`UIStyle.BgDeep`). Country name large at top, subtitle below, horizontal separator, then mood tiles stacked vertically centered.

**Mood tiles** are generated by Claude (`MoodSystem` prompt) based on the selected country and any dietary constraints. While the API call is in flight a "Thinking…" panel is shown. If the call fails, fallback moods are used: Comforting, Light, Festive, Quick, Rich.

Each mood tile is 320 × 112 px with a curated warm palette (terracotta, sage, deep gold, slate blue, plum cycling by index). Tiles are one-shot — committing navigates immediately to `04_Recipe`.

**Back button** — bottom-left, uses `UIStyle.BgCard` dark background.

**State written:** `AppSession.SelectedMood` (string label, e.g. "Comforting").

**LLM call:** `GenerateMoodTiles(country, constraints)` → returns a JSON string array of 4–6 mood words.

### 04 — Recipe

**File:** `Scripts/Scenes/RecipeSceneBootstrap.cs`
**Purpose:** Display the generated recipe and allow ingredient substitutions.

**Layout:** Dark background (`UIStyle.BgDeep`). Two-column layout positioned with manual `anchoredPosition` (no layout group at the root):

| Column | X position | Content |
|--------|-----------|---------|
| Left | −530 px | Ingredients list |
| Right | +200 px | Method steps |

**Recipe header** (top center-left area): dish title in `UIStyle.TextPri`, metadata line (country · servings · time) in `UIStyle.TextSec`, two horizontal separators.

**Ingredients** — Each ingredient is a 460 × 54 px card (`UIStyle.BgCard`). Amount on the left in blue accent, name in `TextPri`. Dwelling on an ingredient opens the **Substitution Panel**.

**Steps** — Each step is a 580 × 60 px card (`UIStyle.BgCardHi`) with a blue step-number pill on the left.

**Substitution Panel** — 370 × 390 px panel anchored at right (720, 50). Shows the target ingredient name, then 3 Claude-generated substitute options as dwell tiles. Committing a substitute replaces the ingredient in `_recipe` and re-renders the whole recipe. A "✕ Keep original" tile dismisses the panel without changes.

**Nav tiles** (bottom corners):
- `← Back to Map` at (−750, −460) — loads `02_Map`
- `↺ New Recipe` at (+750, −460) — clears `CurrentRecipe` and re-runs `LoadRecipe()`

**LLM calls:**
- `GenerateRecipe(country, mood, constraints)` → structured `Recipe` (title, servings, timeMinutes, ingredients[], steps[])
- `GenerateSubstitutions(ingredient, recipeContext)` → string list of 3 substitutes

Both fall back to hardcoded Moroccan Chicken Tagine / stub substitutes on failure.

**State written:** `AppSession.CurrentRecipe`.

## Shared systems

### UIStyle (`Scripts/UIStyle.cs`)
Single static class with all design tokens, procedural sprite generators, and builder helpers used across all scenes.

**Palette:**

| Token | Hex | Usage |
|-------|-----|-------|
| `BgDeep` | `#10131A` | Screen / camera background |
| `BgCard` | `#1A1F2B` | Card / tile background |
| `BgCardHi` | `#252B38` | Elevated card (steps, sub tiles) |
| `Accent` | `#3B82F6` | Fill rings, amounts text, step numbers |
| `Active` | `#3B82F6` | Selected tile highlight |
| `NavBlue` | `#2563EB` | CTA buttons |
| `WarmCue` | `#60A5FA` | Fill ring tint at 60 % dwell |
| `TextPri` | `#F5F1E8` | Primary body text |
| `TextSec` | `#A8A192` | Secondary / meta text |
| `TextMuted` | `#6C6861` | Hints, labels, muted info |
| `Border` | `#2A3142` (70 %) | Card outlines |

**Sprites:** `CardSprite`, `TileSprite`, `ChipSprite`, `PillSprite` — all procedurally generated rounded-rect textures, 9-sliced so they scale without corner distortion.

**Helpers:** `MakeImage`, `MakeText`, `CardBorder`, `PillBorder`, `FillRing`, `Separator`, `HintBar`.

> `CardBorder` and `PillBorder` attach a stretch-anchored overlay Image with `LayoutElement.ignoreLayout = true` — this prevents parent `VerticalLayoutGroup` / `HorizontalLayoutGroup` from treating the border overlay as a zero-height content item (which would collapse all siblings).

---

### DwellTile (`Scripts/Input/DwellTile.cs`)
`MonoBehaviour` + `IPointerEnterHandler` / `IPointerExitHandler`. Requires an `Image` component on the same GameObject.

Key properties:

| Property | Purpose |
|----------|---------|
| `tileId` | Unique string; must match what `MouseDwellInput` broadcasts |
| `fillRing` | Child `Image` set to Filled / Radial360 — driven each Update |
| `autoRearm` | `true` = resets to idle after commit (toggles). `false` = stays committed (nav). |
| `onCommitted` | `UnityEvent` fired once threshold is reached |

**Important:** always call `tile.SetIdleColor(c)` **after** `AddComponent<DwellTile>()`. `OnEnable` runs synchronously during `AddComponent` and resets the background to the serialized `idleColor` default — calling `SetIdleColor` before that point has no effect.

---

### LLM Debug Overlay (`Scripts/Debug/LLMDebugOverlay.cs`)
Persistent overlay (sorting order 999) present in all scenes. An "AI log" badge sits at the right edge of the screen. Dwell the badge to open/close a scrollable panel showing the last N Claude calls: prompt name, duration (ms), timestamp, user content preview, raw response preview.

All Claude calls are logged via `LLMDebugLog.Add(promptName, userContent, rawResponse, durationMs)` inside `ClaudeRecipeService`.

---

### Session state (`Scripts/Model/AppSession.cs`)
Singleton `MonoBehaviour` (`DontDestroyOnLoad`). Carries all user selections across scene loads.

| Property | Type | Default | Set by |
|----------|------|---------|--------|
| `SelectedConstraints` | `List<string>` | `[]` | Scene 01 |
| `SelectedServings` | `int` | 4 | Scene 01 |
| `SelectedTimePreference` | `string` | "any" | Scene 01 |
| `SelectedSpiceLevel` | `string` | "medium" | Scene 01 |
| `SelectedCountry` | `Country` | null | Scene 02 |
| `SelectedMood` | `string` | null | Scene 03 |
| `CurrentRecipe` | `Recipe` | null | Scene 04 |

---

### AI backend (`Scripts/Services/ClaudeRecipeService.cs`)
Calls `claude-haiku-4-5` via `ClaudeClient`. API key read from `StreamingAssets/appsettings.local.json` (gitignored). Falls through to `CachedFallbackService` on any network/parse error so the app stays usable offline.

Four prompts (loaded from `StreamingAssets/Prompts/`):
- `BlurbSystem` — one-sentence country teaser
- `MoodSystem` — JSON array of 4–6 mood words
- `RecipeSystem` — structured JSON recipe object
- `SubSystem` — JSON array of 3 ingredient substitutes
